BUG FOUND: P5-MMAD-001
================================================================================

ID: P5-MMAD-001
Title: Hash File Duplication in Resume-From Builds
Severity: 🟠 HIGH
Type: Data Integrity / Multi-Module State Management
Status: Documented with root cause analysis

Description:
  When using Maven's -rf (resume-from) flag in multi-module reactor builds,
  the plugin creates hash files in TWO locations:
    1. Intended: <reactor-root>/.test-order/hashes/
    2. Unintended duplicate: <skipped-module>/.test-order/hashes/
  
  This violates the ReactorContext design contract for shared cache management.

Root Cause:
  - Skipped modules don't execute prepare mojo
  - prepare mojo calls ensureSharedDirectories()
  - Shared directory not created at reactor-root
  - Later modules create directories in their own base directories instead
  - Causes duplicate state in multiple locations

Reproduction:
  mvn clean && mvn -rf :service-a test -q
  # Result: Hash files in both reactor-root AND service-a directories

Impact:
  - Data integrity concern
  - Confusing developer experience
  - Potential cache consistency issues

Recommended Fix:
  Ensure ReactorContext.ensureSharedDirectories() is called before ANY
  module processes, regardless of -rf flag or module skipping

Affected Code:
  - ReactorContext.ensureSharedDirectories()
  - AbstractTestOrderMojo.initContext()
  - ChangeDetectionHelper.snapshotHashes()
