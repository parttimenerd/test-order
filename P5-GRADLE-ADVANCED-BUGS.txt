P5-GAD-001: Custom Test Task Types Not Supported
  Severity: HIGH
  Area: Custom Gradle Tasks
  Problem: Only withType(Test.class) configures tasks
  Impact: Custom test runners completely unsupported
  Workaround: Document limitation; advise using standard Test class

P5-GAD-005: Race Condition in Index File Writing
  Severity: HIGH
  Location: TestOrderPlugin.java:939 (aggregateDependencyFiles)
  Problem: map.save(indexFile) without file locking
  Impact: Index file corruption in parallel builds
  Fix: Wrap with PersistenceSupport.withFileLock()

P5-GAD-002: No Multi-Project Build Support
  Severity: HIGH
  Area: Multi-project Builds
  Problem: No inheritance mechanism; no shared configuration
  Impact: Difficult to use in large codebases
  Fix: Implement parent project configuration inheritance

P5-GAD-014: Multi-Project State File Collision
  Severity: HIGH
  Location: TestOrderExtension.java:110 (default convention)
  Problem: All subprojects use same .test-order/state.lz4 file
  Impact: Concurrent writes cause data corruption
  Fix: Use project-specific state files or shared directory with locks

P5-GAD-003: No Parallel Execution Support
  Severity: MEDIUM
  Area: Gradle Parallel Execution
  Problem: Plugin doesn't handle gradle --parallel --workers=N
  Impact: Race conditions on shared files; no documentation
  Fix: Add file locking to all I/O or disable in parallel mode

P5-GAD-006: Fragile BuildSrc Detection
  Severity: MEDIUM
  Location: test-order-init.gradle:29
  Problem: Uses string.contains('buildSrc') instead of proper path matching
  Impact: Plugin applied to unintended projects
  Fix: Use proper path matching via File.getParent().getFileName()

P5-GAD-007: Silent Fallback When Invalid Mode Specified
  Severity: MEDIUM
  Area: Gradle Plugins Block
  Problem: Invalid modes silently ignored; falls back to auto-detect
  Impact: Typos like -Dtestorder.mode=lern not caught
  Fix: Add validation; throw exception for invalid modes

P5-GAD-009: Source Root Resolution Falls Back to Hardcoded Path
  Severity: MEDIUM
  Location: TestOrderPlugin.java:906
  Problem: Falls back to "src/main/java" without validation
  Impact: Custom SourceSet configurations ignored
  Fix: Throw error if fallback doesn't exist

P5-GAD-012: Unhandled NumberFormatException in Property Parsing
  Severity: MEDIUM
  Location: TestOrderPlugin.java:1150, 1155
  Problem: Integer.parseInt(override) without try-catch
  Impact: Invalid -Dtestorder.select.topN=invalid crashes build
  Fix: Add try-catch with helpful error message

P5-GAD-013: Dashboard Tasks Missing Input/Output Declarations
  Severity: MEDIUM
  Location: testOrderDashboard and testOrderServe registration
  Problem: No @InputFile annotations or task dependencies declared
  Impact: Gradle can't optimize task ordering; manual ordering required
  Fix: Add inputs.file() declarations

P5-GAD-016: Hardcoded Kotlin Source Path
  Severity: MEDIUM
  Location: TestOrderPlugin.java:568
  Problem: Hardcodes "src/main/kotlin" instead of using SourceSetContainer
  Impact: Custom Kotlin SourceSet directories not supported
  Fix: Resolve via SourceSetContainer like Java sources

P5-GAD-017: TestNG Detection Race Condition
  Severity: MEDIUM
  Location: TestOrderPlugin.java:84, 120
  Problem: Two separate afterEvaluate callbacks; ordering not guaranteed
  Impact: TestNG support may not be added before test configuration
  Fix: Consolidate into single afterEvaluate block
