================================================================================
ROOT CAUSE ANALYSIS
================================================================================

Systematic Issue: No IDE Integration Layer

test-order was designed for Maven/Gradle CLI, not IDEs.

Assumptions that fail in IDE:
  ❌ System properties set by build tool
  ❌ Working directory = project.basedir
  ❌ Classpath constructed by build tool
  ❌ Single instrumentation agent
  ❌ Configuration from properties files

Result: IDE execution completely broken despite CLI working perfectly

SEVERITY ASSESSMENT
================================================================================

OVERALL: 🔴 NOT PRODUCTION READY FOR IDE USAGE

This phase revealed that while test-order works well for CLI/Maven,
it is fundamentally broken for IDE execution - the primary workflow
for most developers.

The 10 bugs represent systematic architectural gaps, not minor issues.
IDE adoption would face:
  - Silent failures (orderer disabled with no warning)
  - Impossible debugging (debugger conflicts)
  - Lost optimization (cache not persisted)
  - Configuration difficulty (manual system properties)

For development teams:
  - CLI/Gradle: Use test-order (works well)
  - IDE: Don't use test-order (broken in IDE context)
