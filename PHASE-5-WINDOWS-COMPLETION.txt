## Detailed Bug Analysis

### Most Critical: Javaagent Path Handling

**Problem:** Both Maven and Gradle plugins construct javaagent arguments without proper shell quoting.

**Affected Files:**
- `test-order-gradle-plugin/src/main/java/.../TestOrderPlugin.java:245`
- `test-order-maven-plugin/src/main/java/.../AbstractTestOrderMojo.java:489`

**Root Cause:**
```java
// BROKEN on Windows with spaces:
"-javaagent:" + agentJar.getAbsolutePath() + "=" + agentArgs
// Output: -javaagent:C:\Program Files\test\agent.jar=...
//         Shell sees: -javaagent:C:\Program
//                    Files\test\agent.jar=...
```

**Impact:** ALL Windows projects in `C:\Program Files\` will fail

---

### Most Pervasive: Line Ending Issues

**Problem:** Code splits on hardcoded `\n` without accounting for CRLF on Windows.

**Affected Files:**
- LineDiff.java (2 locations)
- SourceFileModel.java (1 location, affects parsing)

**Root Cause:**
```java
String[] lines = text.split("\n", -1);
// On Windows CRLF: results in ["line1\r", "line2\r", ...]
```

**Impact:** Structural analysis, change detection, method parsing all affected

---

### Most Insidious: Git Path Separators

**Problem:** Windows filesystem paths with backslashes are passed to git commands expecting forward slashes.

**Affected Files:**
- StructuralDiff.java:56 (git cat-file)
- GitChangeDetector.java:100 (git show)

**Root Cause:**
```java
String relativePath = gitRoot.relativize(absFile).toString();
// Windows: src\main\Foo.java
// Git needs: src/main/Foo.java
```

**Impact:** Change detection, structural diffs, file history all fail
